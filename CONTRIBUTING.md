# Contribution Guidelines

We welcome contributions! To ensure smooth collaboration, please follow the guidelines below:

## How to Contribute

1. Fork the repository.
2. Create a new branch for your changes: `git checkout -b feature-name`.
3. Make your changes.
4. Follow the project's coding standards.
5. Commit your changes: `git commit -m 'Add feature description'`.
6. Push your changes: `git push origin feature-name`.
7. Open a pull request to the main branch.

## Reporting Issues

If you encounter any bugs or have problems, please follow the instructions below to report them:

1. Check if the issue has already been reported in the Issues section.
2. If not, create a new issue and describe the bug clearly, including:
   - Steps to reproduce
   - Expected result
   - Actual result
   - Environment (OS, browser version, etc.)

## Suggesting Features

We encourage feature requests! If you have a new feature to suggest, create an issue and describe your idea in detail, including the problem it solves or the benefit it adds to the project.

## Code of Conduct

Please follow the [Contributor Covenant Code of Conduct](https://www.contributor-covenant.org/) while interacting in the project community.

## Review Process

All pull requests will be reviewed by the project maintainers. We strive to merge contributions that follow the guidelines, are well-tested, and improve the project.
